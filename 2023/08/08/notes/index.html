<load-on-startup>

启用 <load-on-startup> 与不启用的区别

  • 未启用<load-on-startup>，第一次对 servlet 的访问，servlet 将参与 servlet 加载、servlet 实例化、servlet 初始化和请求处理，时间开销比后续的访问大得多。

  • 启用<load-on-startup>，第一次对 servlet 的请求，时间开销可以做到与后续请求相等。对应地，Servlet 容器会在服务器启动期间或在 Web 应用程序部署期间执行 servlet 加载、实例化和初始化。

    • 不必在每个 Servlet 组件上启用 <load-on-startup>。
    • 应该在部署后立即请求的此类 Servlet 组件上启用<load-on-startup>，例如：
      • 提供主菜单/主页的 Servlet 组件。

Servlet 对象创建时间？

servlet 容器创建 Servlet Class 对象有两种情况。

  • 未启用 <load-on-startup>

当向该 servlet 组件发出第一个请求时。

  • 重新启动 Web 应用程序
  • 重新启动 Web 服务器
  • 重新加载 Web 应用程序
  • 重新部署 Web 应用程序

向 servlet 发出第一个请求时。servlet 容器仅在收到对 servlet 的第一个请求时才创建 servlet 类对象，这称为 servlet 的延迟或延迟实例化。

  • 启用 <load-on-startup>

servlet 对象在 web 服务器启动期间或在 Web 应用程序的部署期间加载,称为 servlet 的预初始化或早期初始化。

servlet 对象销毁时间？

Servlet 容器在以下情况下会破坏 servlet 类对象：

  1. 当 Web 应用程序停止/重新加载/取消部署时。
  2. 服务器停止/崩溃时。
  3. 如果 Servlet Class 对象长时间连续空闲。

<load-on-startup> 优先级

多个启用了<load-on-startup>的 servlet，它们的预实例化和预初始化顺序将根据 <load-on-startup> 优先级值确定。

这里 1 是启动时加载优先级值。优先加载值可以是正数、负数或零。

  1. 高正优先级表示低优先级。
  2. 低正优先级表示高优先级。
  3. 负优先级等于不启用<load-on-startup>。

为了更好地理解，将优先级值假设为排名系统，其中 1 的优先级高于 2。

小服务程序优先价值实例化顺序
110 第四
21 第二
3-5 忽略
40 第一
5-10 忽略
62 第三
720 第五
Tomcat 8 以后，空的 `` 没有优先级值会产生错误。

所有 servlet 组件优先级相同时会发生什么？

在这种情况下，所有 servlet 组件都将被视为没有加载启动优先级值，并且这些 servlet 类的对象将在给出第一个请求时创建。这意味着 Servlet 容器将它们视为 <load-on-startup> 未启用。

AbstractAnnotationConfigDispatcherServletInitializer

用于配置容器。Spring中提供接口的实现类ApplicationInitializer，它反过来会查找实现WebApplicationInitializer的类，将配置的任务交给他们来完成。

AbstractAnnotationConfigDispatcherServletInitializer 会同时创建 DispatcherServlet 和 ContextLoaderListener。是传统web.xml方式的替代方式。

注意：

DispatcherServlet启动时，会创建 Spring 应用上下文并加载配置文件或配置类中声明的bean；在Spring web应用中，通常还有由 ContextLoaderListener 创建的另一个上下文。

DispatcherServlet 加载包含 Web 组件的 bean，如控制器、视图解析器以及处理器映射；
ContextLoaderListener 加载应用中的其它bean，通常指驱动应用后端的中间层和数据层组件。

springMVC

@ResponseBody 可以将返回对象json化
@RestController = @ResponseBody + @Controller
@EnableWebMvc 将传入的json数据转对象，同时将这个注解修饰的类注册为 spring bean
@RequestParam 如果请求数据在url中，那么这个注解会将url中的键值对数据赋值给参数
@RequestBody 如果请求数据在请求体中，需要通过这个注解拿到数据
@DataTimeFormat 传入的是时间，可以将这个时间字符串解析成参数能够识别的格式
