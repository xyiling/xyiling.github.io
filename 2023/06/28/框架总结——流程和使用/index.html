Spring

spring全流程

  1. 配置：应用程序通常通过 XML 文件、Java 配置类或注解来配置 Spring 容器。配置文件中包含了应用程序所需的 Bean 的定义以及它们之间的依赖关系等信息。
  2. 启动：应用程序启动时，Spring 容器会根据配置文件中的定义自动创建并初始化所有的 Bean 对象。这些 Bean 对象会被放置在 Spring 容器中，以便在应用程序中使用。
  3. 生命周期管理：Spring 框架提供了一个完整的 Bean 生命周期管理机制，包括初始化、销毁等操作。开发人员可以通过实现特定的接口或使用特定的注解来控制 Bean 的生命周期。参考：spring创建Bean的流程以及Bean的生命周期
  4. 依赖注入：Spring 框架采用依赖注入（Dependency Injection，DI）的方式来管理 Bean 之间的依赖关系。在 DI 中，容器会自动将所需的依赖对象注入到 Bean 中，从而实现 Bean 之间的解耦。
  5. AOP：Spring 框架支持面向切面编程（Aspect-Oriented Programming，AOP），可以通过 AOP 实现对应用程序中的横切关注点的解耦。开发人员可以使用特定的注解或配置文件来定义切面和切点，从而实现对应用程序的控制。
  6. 数据访问：Spring 框架提供了对多种数据访问技术的支持，包括 JDBC、ORM（Object-Relational Mapping，对象关系映射）等。开发人员可以使用 Spring 提供的 API 或集成第三方数据访问框架来实现数据访问。
  7. Web 开发：Spring 框架提供了对 Web 开发的支持，包括 MVC（Model-View-Controller，模型-视图-控制器）架构、RESTful Web 服务等。开发人员可以使用 Spring 提供的 API 或集成第三方 Web 框架来实现 Web 开发。
  8. 测试：Spring 框架提供了对单元测试和集成测试的支持，包括 JUnit、TestNG 等。开发人员可以使用 Spring 提供的 API 或集成第三方单元测试框架来实现测试。
  9. 安全性：Spring 框架提供了对安全性的支持，包括身份验证、授权、加密等。开发人员可以使用 Spring 提供的 API 或集成第三方安全框架来实现安全性。

Bean的生命周期

  1. 实例化

    a. 通过反射推断构造函数，进行实例化

    b. 通过实例工厂、静态工厂，进行实例化

  2. 属性赋值

    a. 解析自动装配
    b. 循环依赖（spring已解决）

  3. 初始化
    a. 调用xxxAware回调方法
    b. 调用初始化生命周期回调（三种）
    c. 如果bean对象实现了aop，则创建动态代理对象

  4. 销毁
    a. spring容器关闭的时候进行调用
    b. 调用销毁生命周期的回调

spring 拓展资料

  1. Bean对象来源

    注解 作用
    @Component 将一个类声明为组件，通常是指该类是一个 Bean，会被 Spring 自动扫描并创建实例。可以通过 @ComponentScan 注解或在 XML 配置文件中配置扫描路径来启用组件扫描。
    @Service 将一个类声明为服务，通常是指该类提供了某种业务功能，会被 Spring 自动扫描并创建实例。
    @Controller 将一个类声明为控制器，通常是指该类负责处理 HTTP 请求，会被 Spring 自动扫描并创建实例。
    @Repository 将一个类声明为仓库，通常是指该类负责数据访问，会被 Spring 自动扫描并创建实例。
    @Configuration 将一个类声明为配置类，通常是指该类用于配置 Bean，会被 Spring 自动扫描并创建实例。
    @Bean 将一个方法声明为 Bean 的创建方法，该方法返回一个实例化的对象，该对象会被 Spring 管理并放入容器中。
    @Import 将一个或多个配置类导入到当前配置类中。
    @ImportResource 从 XML 配置文件中导入 Bean 的定义

SpringMVC

参考SpringMVC详细执行流程

问：请求–>dispatcherServlet–>HandlerMapping–>Handler–>HandlerMapping–>DispatcherServlet–>HandlerAdatper–>ModelAndView–>DispatcherSevlet–>View–>browser这个SpringMVC流程有没有可以补充的

答：这个Spring MVC流程的步骤已经被描述得相当全面了。不过，以下是一些可能有助于更深入理解该流程的补充信息：

  1. 请求处理链：在DispatcherServlet和Handler之间，可以配置一个或多个拦截器。这些拦截器构成了一个请求处理链，可以在请求到达Handler之前或之后拦截请求，执行一些额外的逻辑。拦截器可以用于身份验证、日志记录、性能监测等。
  2. HandlerAdapter：在Handler和ModelAndView之间，HandlerAdapter负责将Handler的输出转换为ModelAndView对象。Spring提供了多个HandlerAdapter实现，以支持不同类型的Handler（如Controller类、Servlet类、WebSocket处理程序等）。
  3. ViewResolver：在DispatcherServlet和View之间，ViewResolver负责将逻辑视图名称解析为实际的View对象。Spring提供了多个ViewResolver实现，以支持不同类型的视图解析策略（如基于JSP的视图解析、基于Thymeleaf的视图解析等）。
  4. 视图渲染：在View和浏览器之间，视图负责将模型数据渲染为最终的HTML或其他格式，并将其发送回浏览器。视图可以使用JSP、Thymeleaf、FreeMarker等模板引擎来生成HTML。
  5. 异常处理：Spring MVC有一个全局异常处理机制，可以捕获应用程序中的所有异常并将它们转换为友好的错误页面或JSON响应。可以通过@ControllerAdvice注解定义一个全局异常处理器，并使用@ExceptionHandler注解处理特定类型的异常。
