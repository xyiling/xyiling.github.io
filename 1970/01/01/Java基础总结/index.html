内部类

成员内部类

访问修饰符

无、public、protected、private

静态成员

不支持静态声明，包括**静态成员、静态方法、静态代码块**，但支持常量

成员方法

仅支持非静态成员方法

继承和实现

可以继承（抽象）外部类、（抽象）内部类、接口

外部类不能继承**成员内部类**

参考：No enclosing instance of type 错误的解决方法

访问特点

  • **内部类访问外部类：不受权限控制**，可以通过成员名字访问

**特殊情况**：内部类和外部类有同名成员，访问情况如何？
这种情况下，内部类直接访问对应成员，默认访问的是内部类的成员，这与变量的局部性原理类似。如果要访问外部类同名成员，可以使用 $外部类名.this.同名成员$ 来实现。

**极端情况：内部类成员变量、外部类成员变量和局部变量**全重名，访问情况？

  • **外部类访问内部类**：内部类是外部类的一个成员，外部类非静态方法访问内部类属性，需要创建内部类对象，通过内部类对象访问，但是如果访问的是内部类静态变量，则可以直接类名.属性；外部类静态方法，由于这类方法不具有对象，因此需要创建对象（内部类对象和外部类对象都需要）才能访问。

静态内部类

静态内部类不需要new，本身不存储任何类的引用，访问内部类或外部类或外部其他类，都需要new对象，然后访问，且访问收到权限修饰符的限制。
静态内部类成员特点：普通类具有的，在静态类也都有。
静态内部类理论上不算是内部类，它只是嵌套在外部类内部，不依赖外部类；成员内部类依赖外部类而存在，创造成员内部类的引用，必须外部类已存在。
